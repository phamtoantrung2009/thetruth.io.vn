# Architecture

## System Overview

```
Developer → GitHub → CI/CD → Static Site → Cloudflare CDN → User
                                    ↓
                            Email Worker + D1 + KV
```

## Philosophy

**Static-first, minimal complexity.**

- All content is static HTML
- No server-side rendering
- No database queries at runtime
- Build-time generation of all artifacts

## Core Flow

```
content/*.md (Markdown)
        ↓
scripts/build.py (Python)
        ↓
_site/ (Static HTML)
        ↓
Cloudflare Pages
        ↓
CDN Delivery
```

## Components

### 1. Content Layer

**Location:** `content/`

- Markdown files with YAML frontmatter
- Fields: title, slug, excerpt, date, tags

### 2. Build Layer

**Location:** `scripts/build.py`

- Parse markdown content
- Render Jinja2 templates
- Generate search index, sitemap, RSS
- Generate OG images via Playwright

### 3. Template Layer

**Location:** `layouts/`

- `index.html` - Homepage
- `post.html` - Blog post
- `articles.html` - Article list
- `about.html` - About page
- `tensions.html` - Tension cards

### 4. Deployment Layer

**Location:** `.github/workflows/build.yml`

- Triggered on git push
- Runs tests
- Builds static site
- Deploys to Cloudflare Pages

### 5. Worker Layer

**Location:** `workers/subscribe/`

- Email subscriptions
- Double opt-in flow
- Rate limiting via KV
- Stores subscribers in D1

## Data Flow

### Static Content

```
User Request
    ↓
Cloudflare CDN (_site/)
    ↓
Static HTML (no processing)
```

### Email Subscription

```
User submits email
    ↓
POST /subscribe (Worker)
    ↓
Validate + Rate Limit (KV)
    ↓
Create pending record (D1)
    ↓
Send confirmation (Resend)
    ↓
User clicks confirmation
    ↓
Worker activates record
    ↓
Status = active
```

## Artifact Generation

All artifacts generated at build time:

| Artifact | Generated By | Purpose |
|----------|--------------|---------|
| `*.html` | build.py | Blog pages |
| `sitemap.xml` | build.py | Search engines |
| `robots.txt` | build.py | Crawler control |
| `rss.xml` | build.py | RSS readers |
| `search.json` | build.py | Client search |
| `site-manifest.json` | build.py | AI pipeline |
| `posts.json` | build.py | Content index |
| `og/*.png` | Playwright | Social cards |

## Design Principles

### DO

- Keep it static
- Generate at build time
- Use Cloudflare edge
- Cache aggressively

### DON'T

- No server-side rendering
- No runtime database queries
- No tracking scripts
- No unnecessary JavaScript

## Scalability

The static architecture scales automatically:

- CDN serves all content
- No server to scale
- Worker handles only subscription
- D1 + KV for subscriber data only
